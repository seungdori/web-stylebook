> TERMINAL-FIRST BRAND PAGE

A style that excels for developer-targeted products. The key is not just a green screen, but embedding command patterns and status output narratives into the UI structure.

개발자 대상 제품에 강한 스타일입니다. 단순 녹색 화면이 아니라, 명령어 패턴과 상태 출력 같은 내러티브를 UI 구조에 넣는 것이 핵심입니다.

開発者向け製品に強いスタイルです。単なる緑色の画面ではなく、コマンドパターンやステータス出力のようなナラティブをUI構造に組み込むことがポイントです。

$ init layout --mode command-log
$ set typography --family mono --weight strong
$ deploy cta --position final --contrast high

AI Request Prompt

Design a SaaS landing page for developers in terminal UI style.

Style rules:
- Background: Dark green (#071109) + CRT scanline pattern (repeating-linear-gradient 2px intervals)
- Typography: Full monospace (JetBrains Mono), headings bold and large, body same font
- Structure: Terminal window frame (top bar + main screen), sections expressed as command logs
- Components: Console blocks ($ prompt + output) pattern instead of cards
- Colors: Green text (#b9ffbe) + dark green lines (#1e4f22), accent limited to lime (#d4ff5c)
- Motion: Typing/sequential output feel (translateY 10px, 0.6s ease, 0.08s intervals)

Responsive:
- Mobile: Console block font size reduced, padding adjusted
- Desktop: max-width 1040px, console blocks in vertical stack

Restrictions:
- No gradient backgrounds or blur effects
- No mixing sans-serif/serif fonts (monospace only)
- No rounded corners exceeding 16px

Output:
1) Design tokens (color/typography/spacing/border)
2) Terminal frame / Hero / Features (console blocks) / Logs / CTA structure
3) HTML/CSS code with responsive support
개발자 대상 SaaS 랜딩 페이지를 터미널 UI 스타일로 설계해줘.

스타일 규칙:
- 배경: 다크 그린(#071109) + CRT 스캔라인 패턴(repeating-linear-gradient 2px 간격)
- 타이포: 전체 모노스페이스(JetBrains Mono), 제목은 굵고 크게, 본문도 동일 폰트
- 구조: 터미널 창 프레임(상단 바 + 메인 스크린), 섹션을 명령어 로그처럼 표현
- 컴포넌트: 카드 대신 콘솔 블록($ 프롬프트 + 출력) 패턴 활용
- 컬러: 그린 텍스트(#b9ffbe) + 다크 그린 라인(#1e4f22), 포인트는 라임(#d4ff5c) 한정
- 모션: 타이핑/순차 출력 느낌(translateY 10px, 0.6s ease, 0.08s 간격)

반응형:
- 모바일: 콘솔 블록 폰트 크기 축소, 패딩 조정
- 데스크톱: max-width 1040px, 콘솔 블록 세로 스택

금지사항:
- 그라데이션 배경이나 블러 효과 사용 금지
- 산세리프/세리프 폰트 혼용 금지(모노스페이스 통일)
- 둥근 코너 16px 초과 금지

출력:
1) 디자인 토큰(컬러/타이포/간격/보더)
2) 터미널 프레임 / Hero / Features(콘솔 블록) / Logs / CTA 구조
3) 반응형 대응이 포함된 HTML/CSS 코드
開発者向けSaaSランディングページをターミナルUIスタイルで設計してください。

スタイルルール:
- 背景: ダークグリーン(#071109)+CRTスキャンラインパターン(repeating-linear-gradient 2px間隔)
- タイポグラフィ: 全体モノスペース(JetBrains Mono)、見出しは太く大きく、本文も同じフォント
- 構造: ターミナルウィンドウフレーム(上部バー+メインスクリーン)、セクションをコマンドログのように表現
- コンポーネント: カードの代わりにコンソールブロック($プロンプト+出力)パターンを活用
- カラー: グリーンテキスト(#b9ffbe)+ダークグリーンライン(#1e4f22)、アクセントはライム(#d4ff5c)に限定
- モーション: タイピング/順次出力の感覚(translateY 10px, 0.6s ease, 0.08s間隔)

レスポンシブ:
- モバイル: コンソールブロックのフォントサイズ縮小、パディング調整
- デスクトップ: max-width 1040px、コンソールブロック縦スタック

禁止事項:
- グラデーション背景やブラー効果の使用禁止
- サンセリフ/セリフフォントの混用禁止(モノスペース統一)
- 角丸コーナー16px超過禁止

出力:
1) デザイントークン(カラー/タイポグラフィ/スペーシング/ボーダー)
2) ターミナルフレーム / Hero / Features(コンソールブロック) / Logs / CTA構造
3) レスポンシブ対応を含むHTML/CSSコード
